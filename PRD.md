# Product Requirements Document

## Project Title

Websale: Ecommerce Website for Selling Website Packages

## Project Type

College training project built strictly with Django for backend and Django HTML templates for frontend.

## 1. Problem Statement

Many small businesses, students, and freelancers need websites but do not always know where to start. This project solves that problem by presenting website development packages in a simple ecommerce-style platform where users can view offerings, compare options, and contact the seller.

## 2. Project Goal

To build a basic ecommerce-style website in Django that promotes and sells website packages such as business websites, portfolio websites, and online store setups.

## 3. Objectives

- Build the project using Django framework only.
- Use Django views, URLs, templates, and static files correctly.
- Create a clean multi-page website with reusable layout.
- Demonstrate how ecommerce features can be added in phases.
- Prepare the foundation for future modules like products, cart, checkout, and admin management.

## 4. Target Users

- Small business owners
- Students
- Freelancers
- Startup founders
- Local shops wanting an online presence

## 5. Proposed Features

### Phase 1: Basic Project Scope

- Home page with project introduction
- About page explaining the concept
- Services page listing website packages
- Contact page for customer enquiry details
- Shared navigation bar and footer
- Static CSS styling

### Phase 2: Future Enhancements

- Product/package model in Django
- Dynamic package listing from database
- User registration and login
- Cart system
- Checkout flow
- Order tracking
- Admin dashboard for managing packages and enquiries

## 6. Functional Requirements

- The system should display a home page.
- The system should display an about page.
- The system should display a services page.
- The system should display a contact page.
- Navigation links should allow users to move between pages.
- The layout should be reusable using a base template.
- Static files should load properly for styling.

## 7. Non-Functional Requirements

- The website should be simple and easy to use.
- The interface should be responsive for desktop and mobile.
- The project should follow Django project structure.
- The code should be readable for training and evaluation.

## 8. Technology Stack

- Backend: Django
- Frontend: HTML, CSS, Django Template Language
- Database: SQLite
- Development Tool: Python virtual environment

## 9. Main Pages

- Home
- About
- Services
- Contact

## 10. Sample User Flow

1. User visits the home page.
2. User checks available website packages on the services page.
3. User reads about the project or company on the about page.
4. User visits the contact page to make an enquiry.

## 11. Expected Outcome

At the end of development, the project will demonstrate a working Django multi-page ecommerce-style website that can later be extended into a complete service-selling platform.

## 12. Conclusion

Websale is a practical Django training project because it covers the most important beginner-level concepts while leaving enough scope for future expansion into a full ecommerce application.
